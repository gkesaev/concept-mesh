# ConceptMesh

## Vision

ConceptMesh is a visual exploration platform where knowledge is a living, infinite mesh. You see concepts as cards floating in space, connected by glowing threads. Zoom out and you see galaxies of domains — Mathematics, Physics, Computer Science. Zoom in and a concept fractures into deeper sub-concepts. Click a card and an AI-generated interactive visualization comes alive in a focused window, teaching you through play.

The graph never ends. Every concept is a door to more concepts. A serendipity engine whispers unexpected connections: *"You might not expect this, but binary search connects to musical harmony because both are about dividing intervals to find resonance."*

This is not a textbook. It is not a course. It is an exploration tool for the curious.

## How It Works

### The Mesh
An infinite 2D canvas filled with concept cards. Cards are connected by color-coded threads that represent relationships. The layout is physics-based — related concepts cluster together, domains form galaxies, and the whole thing feels organic and alive.

The canvas is scrollable and zoomable:
- **Zoomed out**: You see domain clusters — large labeled regions with tiny dots for concepts
- **Mid zoom**: Concept cards show their name and domain, connections become visible
- **Zoomed in**: Full card detail — description, status, connections. Cards you can click to explore.

### Concept Cards
Each card represents a single concept. Cards have states:
- **Unexplored**: Dimmer, slightly pulsing. The concept exists but hasn't been visualized yet. Click to generate.
- **Explored**: Vibrant, with a subtle glow. Has an AI-generated interactive visualization ready to view.
- **Generating**: Animated, showing the AI pipeline at work.

Cards display:
- Concept name
- Domain badge
- Brief explanation
- Connection count
- Exploration status

### Connections
Lines between cards representing relationships. For now, all connections are type "related" — this will expand later to include prerequisite, application, contrast, analogy, etc.

Each connection can have a `reason` — a brief explanation of why these two concepts are connected. AI-generated connections always have reasons.

### Interactive Visualizations
When you click a concept card, a modal opens with an AI-generated interactive visualization. These are real, runnable React components generated by Claude through a multi-shot pipeline:

1. **Plan**: AI analyzes the concept and designs a visualization strategy (what to show, what controls to offer, what insight to reveal)
2. **Generate**: AI writes the actual React.createElement code with canvas graphics, sliders, buttons, animations
3. **Validate**: The code is checked locally for syntax/runtime errors
4. **Fix**: If validation fails, AI gets one chance to fix the code
5. **Save**: Working visualization is saved to the database for instant replay

Visualizations run in an iframe sandbox for safety. They have access to React hooks, Canvas API, and Math — nothing else.

### Concept Expansion
When you're viewing a concept, you can expand it — the system generates sub-concepts and new connections, adding them to the mesh around the parent card. This is how the graph grows organically.

For example, expanding "Neural Networks" might spawn: "Backpropagation", "Activation Functions", "Loss Functions", "Gradient Descent" — each positioned near the parent, each with connections to each other and to existing concepts in the mesh.

### Serendipity Engine
The most exciting feature. The system finds concepts that are semantically similar (their descriptions are close in vector embedding space) but graph-distant (far apart in the mesh, few or no connections between them). It then asks Claude to articulate why they're connected.

This produces genuine surprises:
- "Fractals connect to Stock Markets because both exhibit self-similar patterns across scales"
- "Entropy connects to Urban Planning because both deal with disorder emerging from local optimization"
- "Binary Search connects to Musical Harmony because both divide intervals to find target values"

These connections appear as glowing new threads in the mesh, with a banner explaining the discovery.

## User Experience

### First Visit
You land on the mesh with 5 seed concepts floating in space: Binary Search, Derivatives, Neural Networks, Probability, Velocity. They're connected by a few threads. The canvas is dark, spacious, inviting.

You zoom in on one. The card expands to show its description. You click it. A modal opens — the AI starts generating. You see progress: "Planning visualization... Generating code... Validating..." Then an interactive visualization appears. You play with it.

You close the modal. You notice the card is now glowing. You see its connections — some lead to unexplored nodes at the edges. You click one. A new card appears, fading in. The AI generates its visualization. The mesh grows.

A serendipity banner slides in: "Did you know? Derivatives connect to Neural Networks because backpropagation is literally the chain rule applied repeatedly." You click it and both concepts highlight, a new glowing thread between them.

You keep pulling threads. The mesh grows. Two hours pass.

### Mobile
Same experience, optimized for touch:
- Pinch to zoom, swipe to pan
- Tap a card to select, double-tap to open modal
- Modal is full-screen on mobile
- Canvas controls at the bottom

## Technical Boundaries

- All AI generation happens server-side (API key never reaches the client)
- Generated visualization code runs in an iframe sandbox
- Viewport-based loading — only fetch concepts near the camera
- Force-directed layout runs in a Web Worker (no UI jank)
- All dependencies are MIT/ISC/Apache-2.0 licensed (commercially viable)
- PostgreSQL with pgvector for semantic search
- Next.js 15 with App Router for the full-stack framework

## What This Is NOT

- Not a course platform with lessons and quizzes
- Not a note-taking app or knowledge base
- Not a curated, hand-built visualization library
- Not limited to any single domain
- Not a social network (yet — multi-user is planned, social is not)

## Success Criteria

The project is successful when:
1. You can spend an hour exploring without running out of things to discover
2. The serendipity engine regularly produces "wait, really?" moments
3. The visualizations actually help you understand concepts, not just look pretty
4. The mesh feels alive and organic, not like a database viewer
5. It works smoothly on both desktop and mobile
6. Someone who sees it over your shoulder says "what IS that?"
